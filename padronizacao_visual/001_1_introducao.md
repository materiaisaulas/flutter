#### Padronização visual 

A construção de aplicações em Flutter vai além da simples utilização de widgets. À medida que os sistemas evoluem, torna-se fundamental adotar uma organização de código que favoreça a manutenção, a escalabilidade e a clareza estrutural. Por esse motivo, este material adota desde o início uma abordagem profissional, na qual o projeto já nasce estruturado em camadas, mesmo que nem todos os conceitos sejam explorados imediatamente. Essa decisão permite que o estudante desenvolva suas habilidades dentro de um contexto próximo ao encontrado no mercado, evitando a necessidade de reestruturações futuras.

A estrutura do projeto foi organizada com base na separação de responsabilidades, distribuindo os elementos da aplicação em diretórios que refletem suas funções dentro do sistema. No diretório `core`, concentram-se os elementos fundamentais e transversais da aplicação, ou seja, aqueles que não pertencem a uma funcionalidade específica, mas que são utilizados globalmente. Dentro dele, o subdiretório `layout` contém o arquivo `app_scaffold.dart`, responsável por definir a estrutura base das telas, incluindo elementos como barra superior, menu lateral e área de conteúdo. Já o subdiretório `theme` abriga o arquivo `app_theme.dart`, onde são definidas as configurações visuais globais, como cores e padrões de estilo, permitindo a padronização da interface.

O diretório `features` é responsável por agrupar as funcionalidades da aplicação. Cada funcionalidade é organizada em seu próprio espaço, facilitando a evolução do sistema de forma modular. No exemplo atual, a funcionalidade inicial está representada pelo diretório `home`, que contém o arquivo `home_page.dart`. Esse arquivo define a tela principal da aplicação e será o ponto de partida para a construção e visualização dos componentes estudados.

No diretório `shared`, encontram-se os elementos reutilizáveis que podem ser utilizados por diferentes partes da aplicação. Dentro dele, o subdiretório `widgets` abriga componentes compartilhados, como o `app_drawer.dart`, responsável pelo menu lateral. Essa separação evita duplicação de código e centraliza a manutenção de elementos comuns.

Por fim, o arquivo `main.dart`, localizado na raiz do diretório `lib`, atua como ponto de entrada da aplicação. Ele é responsável por inicializar o sistema, configurar o tema global, definir as rotas e aplicar configurações gerais de layout. A partir dele, toda a estrutura previamente organizada é integrada, permitindo que a aplicação funcione de forma coesa.

Essa organização estabelece uma base sólida para o desenvolvimento, permitindo que novos componentes e funcionalidades sejam adicionados de maneira estruturada. Ao compreender essa divisão desde o início, o estudante passa a enxergar o projeto não apenas como um conjunto de arquivos, mas como um sistema organizado em camadas, no qual cada parte possui um papel bem definido.
