#### main.dart 

O arquivo `main.dart` representa o ponto de entrada da aplicação Flutter e tem como principal objetivo inicializar o sistema e definir sua configuração global. A função `main()` executa o método `runApp`, que recebe como parâmetro o widget raiz da aplicação, neste caso a classe `MyApp`. Essa classe herda de `StatelessWidget`, indicando que não possui estado interno mutável, e sua responsabilidade é construir a estrutura principal da aplicação por meio do método `build`. Dentro desse método, é retornado um `MaterialApp`, que funciona como o contêiner global do aplicativo, sendo responsável por aspectos como tema, navegação, título e configurações gerais de interface.

O `MaterialApp` está configurado com propriedades importantes. A propriedade `title` define o nome da aplicação, utilizado pelo sistema operacional em alguns contextos. A propriedade `debugShowCheckedModeBanner` foi definida como `false` para remover o banner de debug exibido por padrão durante o desenvolvimento. Já a propriedade `theme` recebe `AppTheme.light`, indicando que as definições visuais da aplicação foram centralizadas em um arquivo separado, promovendo padronização e organização. A propriedade `initialRoute` define a rota inicial da aplicação, enquanto `routes` estabelece um mapeamento entre caminhos e páginas, permitindo a navegação entre telas. Nesse caso, a rota `'/'` está associada à `HomePage`, que será exibida ao iniciar o aplicativo.

Um dos pontos mais relevantes desse código está na propriedade `builder` do `MaterialApp`. Esse `builder` permite interceptar a construção de todas as telas da aplicação, aplicando uma camada de layout global. Isso significa que qualquer página renderizada passará por essa estrutura, garantindo consistência visual e comportamental. O parâmetro `child` representa o conteúdo da tela atual (definido pelas rotas), e ele será encapsulado pelos widgets definidos dentro do `builder`.

O widget `SafeArea` é utilizado para garantir que o conteúdo da aplicação não seja exibido em áreas não seguras da tela, como regiões ocupadas por elementos do sistema operacional (notch, barra de status, bordas curvas, entre outros). Ele ajusta automaticamente o espaçamento do conteúdo para que tudo seja exibido dentro da área visível e segura do dispositivo, sendo especialmente importante em dispositivos modernos com diferentes formatos de tela.

O `LayoutBuilder` é um widget que permite construir a interface com base nas restrições de tamanho disponíveis no momento da renderização. Ele fornece, por meio do parâmetro `constraints`, informações como largura e altura máximas disponíveis. No código apresentado, ele é utilizado para implementar uma lógica de responsividade: caso a largura disponível (`constraints.maxWidth`) seja maior que 900 pixels, a aplicação limita o conteúdo a esse valor; caso contrário, utiliza a largura total disponível. Isso é fundamental para adaptar a aplicação a diferentes dispositivos, como celulares, tablets e navegadores web.

O widget `Center` tem a função de centralizar seu filho dentro do espaço disponível. Ele garante que o conteúdo fique posicionado no centro da tela (tanto horizontal quanto verticalmente), contribuindo para uma apresentação visual mais equilibrada, especialmente quando a largura da aplicação é limitada artificialmente, como ocorre neste caso.

O `ConstrainedBox` é responsável por impor restrições adicionais ao tamanho de seu filho. No código, ele utiliza `BoxConstraints(maxWidth: maxWidth)` para limitar a largura máxima do conteúdo da aplicação. Isso evita que a interface se estique excessivamente em telas muito largas, mantendo uma aparência mais organizada e próxima de aplicações profissionais. Esse padrão é bastante utilizado em sistemas web e desktop, onde a largura da tela pode variar significativamente.

Por fim, a propriedade `routes` define um mapa de navegação baseado em rotas nomeadas. Cada chave do mapa representa um caminho (string), e cada valor é uma função que retorna o widget correspondente àquela rota. Esse mecanismo permite desacoplar a navegação da implementação das telas, facilitando a manutenção e a escalabilidade da aplicação.


```dart
// lib/main.dart

import 'package:flutter/material.dart';
import 'core/theme/app_theme.dart';
import 'features/home/home_page.dart';

void main() {
  runApp(const MyApp());
}

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'App Padrão Flutter',
      debugShowCheckedModeBanner: false,
      theme: AppTheme.light,
      builder: (context, child) {
        return SafeArea(
          child: LayoutBuilder(
            builder: (context, constraints) {
              final maxWidth = constraints.maxWidth > 900
                  ? 900.0
                  : constraints.maxWidth;
              return Center(
                child: ConstrainedBox(
                  constraints: BoxConstraints(maxWidth: maxWidth),
                  child: child,
                ),
              );
            },
          ),
        );
      },
      initialRoute: '/',
      routes: {'/': (context) => const HomePage()},
    );
  }
}

```
